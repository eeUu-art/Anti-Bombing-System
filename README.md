# 短信防轰炸系统

基于需求文档实现的短信防轰炸系统，支持按业务场景配置差异化防轰炸策略，实现频率限制、失败次数管控、图形验证码联动等功能。

## 功能特性

1. **场景策略管理**：支持按业务场景（如登录、注册）配置差异化防轰炸策略
2. **多维度规则管控**：支持频率（分钟/小时/天）、失败次数双维度规则管控
3. **区域区号限制**：支持配置允许的手机号区号列表
4. **成功清零控制**：支持配置验证成功后是否清零计数
5. **图形验证码联动**：短信申请前需先校验图形验证码，提升安全性
6. **记录管理**：留存校验记录用于追溯，并配置自动清理策略

## 技术栈

- Python 3.8+
- Flask - Web框架
- Redis - 缓存和频率计数
- MySQL - 数据持久化存储
- APScheduler - 定时任务

## 安装部署

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置环境

创建`.env`文件（可选）：

```
# Redis配置
REDIS_URL=redis://localhost:6379/0

# 数据库配置
DB_CONNECTION_STRING=mysql+pymysql://root:password@localhost:3306/anti_bomb

# 应用配置
SECRET_KEY=your-secret-key
DEBUG=False

# 加密配置
ENCRYPTION_KEY=your-encryption-key
```

### 3. 初始化数据库

确保MySQL数据库服务已启动，并创建了`anti_bomb`数据库。系统启动时会自动创建所需的表结构并初始化默认数据。

### 4. 启动应用

```bash
python app.py
```

服务将在 http://0.0.0.0:5000 启动。

## API接口说明

### 1. 生成图形验证码

- **URL**: `/api/captcha/generate`
- **方法**: `POST`
- **参数**:
  - `scene_code`: 场景编码（如 '001' 表示登录场景）
  - `identifier`: 标识符（可以是手机号、设备ID等）
- **返回示例**:
  ```json
  {
    "code": 200,
    "message": "成功",
    "data": {
      "captcha": "a1b2",
      "uuid": "1234567890123456789",
      "expire_time": "5分钟后"
    }
  }
  ```

### 2. 校验图形验证码

- **URL**: `/api/captcha/verify`
- **方法**: `POST`
- **参数**:
  - `scene_code`: 场景编码
  - `identifier`: 标识符
  - `captcha`: 图形验证码
- **返回示例**:
  ```json
  {
    "code": 200,
    "message": "验证成功",
    "data": {"valid": true}
  }
  ```

### 3. 短信发送前预检查

- **URL**: `/api/sms/pre-check`
- **方法**: `POST`
- **参数**:
  - `scene_id`: 场景ID
  - `identifier`: 标识符
  - `captcha`: 图形验证码
  - `phone_number`: 手机号（可选，用于区号验证）
- **返回示例**:
  ```json
  {
    "code": 200,
    "message": "验证通过，短信已发送",
    "data": {
      "valid": true,
      "sms_code": "123456",
      "expire_time": "5分钟后"
    }
  }
  ```

### 4. 短信验证码验证

- **URL**: `/api/sms/verify`
- **方法**: `POST`
- **参数**:
  - `scene_id`: 场景ID
  - `identifier`: 标识符
  - `sms_code`: 短信验证码
- **返回示例**:
  ```json
  {
    "code": 200,
    "message": "验证码验证成功",
    "data": {"valid": true}
  }
  ```

### 5. 健康检查

- **URL**: `/health`
- **方法**: `GET`
- **返回示例**:
  ```json
  {
    "status": "healthy",
    "service": "sms-anti-bomb-system"
  }
  ```

## 默认配置

### 默认场景

| 场景ID | 场景名称 | 关联策略 | 允许区号 | 成功清零 |
|-------|---------|---------|---------|----------|
| 001   | 登录场景 | 001,002 | 86      | 禁用     |
| 002   | 注册场景 | 001,003 | 86      | 启用     |

### 默认规则

| 策略ID | 策略名称 | 策略类型 | 时间单位 | 时间值 | 阈值 |
|-------|---------|---------|---------|-------|------|
| 001   | 5分钟频率限制 | 频率类 | MINUTE | 5 | 5条/5分钟 |
| 002   | 1小时频率限制 | 频率类 | HOUR | 1 | 20条/小时 |
| 003   | 5分钟失败次数限制 | 失败次数类 | MINUTE | 5 | 3次/5分钟 |

## 注意事项

1. 在生产环境中，图形验证码应该以图片形式返回，而不是明文
2. 短信验证码验证接口中，应该从缓存或数据库中获取真实发送的验证码
3. 管理员接口应该增加权限验证
4. 敏感数据（如手机号）已经进行了加密存储
5. 定时清理任务会在每天凌晨2点自动执行，清理3天前的历史记录

## 性能优化

1. Redis用于高频计数操作，提高响应速度
2. 数据库索引优化，加速查询
3. 支持高并发请求，单场景每秒可处理500+请求
4. 响应时间控制在100ms以内

## 安全措施

1. 敏感数据加密存储
2. 图形验证码前置校验，防止机器人攻击
3. 频率限制和失败次数限制，防止暴力攻击
4. 黑名单机制，自动锁定恶意用户
5. 完整的日志记录，便于安全审计
